## About this demo

API Key authentication lets you protect endpoints with a static token sent in the `Authorization` header, with optional role-based access control. KrakenD validates the key against the list declared at service level and rejects any request without a valid key.

## Try it

A successful call (valid key):

```bash
curl -iG -H 'Authorization: Bearer 58427514-be32-0b52-b7c6-d01fada30497' http://localhost:8080/api-key
```

And without a key (returns `401`):

```bash
curl -iG http://localhost:8080/api-key
```
