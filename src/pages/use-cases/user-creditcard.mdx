## Content manipulation with regular expressions

This endpoint calls a backend that returns user information, from which we want to surface only the `credit_card` field. The `modifier/response-body` component applies a regex on the field before returning it, so the actual digits never leave the gateway.

The regex used in this example is `"find": "^.*(\\d{4})"` and we replace the captured group with `XXXX-${1}`, so a string like `1234-56789-1234-5678` becomes `XXXX-5678`. It's an illustrative — not production-grade — example of how to mask sensitive data at the edge.
