## gRPC to REST conversion

![gRPC to REST](https://www.krakend.io/images/documentation/grpc.dark.png)
The `/travel` endpoint connects to two different gRPC services returning flights and trains information. It takes care of data remapping and transformation and it merges the responses from the two services together, returning it as a REST API.

A use case like this simplifies the consumption of internal gRPC services for end-users who don't need to deal with Protobuf directly.

export default function MDXPage({ children }) {
  return <MdxLayout resources={resources}>{children}</MdxLayout>
}
