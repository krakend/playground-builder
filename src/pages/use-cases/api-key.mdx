## Endpoint with API Key access
This endpoint requires the request to provide a valid API Key to access the contents.

The configuration block responsible for this check is:
```json
"auth/api-keys": {
  "roles": [
    "admin"
  ]
}
```

It will require that the API Key belongs to an `admin`. The API keys are declared statically outside the endpoint at the service level.

You can test this endpoint with:

```bash
curl -iG -H 'Authorization: Bearer 58427514-be32-0b52-b7c6-d01fada30497' 'http://localhost:8080/api-key'
```

If you don't pass a valid key, you will get an error:

```bash
curl -iG -H 'Authorization: Bearer invalid' 'http://localhost:8080/api-key'
```
