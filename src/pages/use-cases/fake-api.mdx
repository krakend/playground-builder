## Wildcard routes
The wildcard functionality lets you declare endpoints with many implicit paths in their backend. Instead of expressing all possible routes one by one, you can create one or more endpoints that will respond to a path pattern instead. The gateway then forwards all routes, starting with the configured path and method, to the backend(s).

In this case, any call to `/fake-api/*` will call our fake backend trying to retrieve its contents.

}
