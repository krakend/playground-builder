## Backend-issued payload, gateway-signed token

This use case simulates a monolithic backend that can produce a JSON payload (e.g. a session-like response) but **doesn't know how to sign JWTs**. The `auth/signer` component picks the payload up and signs it cryptographically with HS256 before returning it to the caller.

It's the bridge between a classic session workflow and a JWT-based one — your backends stay as-is, the gateway turns their output into proper signed tokens. The resulting token works with any endpoint that validates it with the same key, like [`/private/custom`](/demo/use-cases/private-custom).
