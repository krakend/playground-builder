## MCP Server — Agentic Gateway for AI tools


This endpoint implements a **Model Context Protocol (MCP) server** so AI
agents (Claude Desktop, Cursor, internal copilots, anything that speaks
MCP) can call your APIs through KrakenD with the same auth, rate limits,
observability and aggregation policies you already use for HTTP traffic.

### About MCP

The Model Context Protocol is an open standard for exposing tools and
data sources to AI models in a uniform way. KrakenD's MCP server turns
gateway endpoints into MCP tools, so the model sees a clean tools list
and the gateway hides protocol heterogeneity, governance, and
aggregation behind it.

### Demo scenario: KYC counterparty risk dossier

This playground exposes a **`kyc-dossier-aggregator`** MCP server with
two tools, both backed by the same fan-out:

- **REST** entity registry → `/__kyc/aggregate/{lei}` resolves the legal
  entity from `mocked_apis` (`http://mocked_apis:8091/kyc/entity/{lei}.json`).
- **GraphQL** ownership chain → `mocked_apis` POST `/graphql` returns
  the UBO chain for the LEI.
- **SOAP** sanctions screening → `mocked_apis` POST `/soap/sanctions`
  returns a sanctions match envelope.

KrakenD aggregates the three responses into a single payload. The
two MCP tools differ only in the output format:

| Tool | Output | Backed by |
|---|---|---|
| `get_kyc_dossier` | Compact **TOON** text optimized for LLM token budgets | `/__kyc/aggregate-toon/{lei}` (the JSON aggregate run through a Lua post-merge transformer in `config/krakend/lua/to_toon.lua`) |
| `get_kyc_dossier_legacy` | Verbose JSON, the raw aggregated structure | `/__kyc/aggregate/{lei}` |

Both take a single argument: `lei` — a 20-char Legal Entity Identifier
(ISO 17442). Three demo entities are pre-loaded:

- `549300CLEAN000000001` — Helvetica Industries AG (clean)
- `549300VOLGA000000001` — Volga Energy Trading LLC (sanctioned)
- `549300MERC0000000001` — Mercator Capital Holdings Ltd (sanctioned UBO buried in a shell chain)

### Try it interactively

The fastest way to see the server end-to-end is the
[Agentic Gateway (MCP) tab in the Interactive Demos](http://localhost:8080/interactive-demo/#mcp-chat).
It runs a chat against Claude Haiku 4.5 wired to this MCP server, with
the four canned prompts on the welcome screen exercising both tools
(including the TOON-vs-JSON token-savings comparison).

### Wire up your own MCP client

Point any MCP-speaking client at:

```http
POST http://localhost:8080/mcp
```

JSON-RPC over HTTP, both `application/json` and `text/event-stream`
Accept headers are honoured. Standard MCP methods (`tools/list`,
`tools/call`) work out of the box.

### Technical Details

- **Method**: POST only
- **Protocol**: Model Context Protocol (JSON-RPC 2.0 over HTTP, with
  optional SSE responses)
- **Backend**: 3-protocol fan-out aggregated at the gateway (REST +
  GraphQL + SOAP), with a Lua transformer for the TOON variant
- **License**: Enterprise Edition only

See the [AI Gateway documentation](https://www.krakend.io/docs/enterprise/ai-gateway/)
and the [MCP Server documentation](https://www.krakend.io/docs/enterprise/ai-gateway/mcp/)
for more.
